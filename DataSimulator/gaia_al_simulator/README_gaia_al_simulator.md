# Gaia-like AL astrometry simulator

This script generates a synthetic Gaia-like along-scan (AL) astrometric data set from a GOST scanning-geometry CSV. It is intended for orbit-fitting tests and controlled recovery experiments.

It does **not** simulate Gaia CCD crossing times or the full Gaia calibration chain. The generated quantity is a local tangent-plane AL observable:

```text
AL = (Delta alpha* + mu_alpha* dt + orbital_Delta alpha*) sin(psi)
   + (Delta delta  + mu_delta  dt + orbital_Delta delta ) cos(psi)
   + parallax * parallaxFactorAlongScan
   + noise
```

Here `psi` and `parallaxFactorAlongScan` come from GOST. The AL value itself is generated by this script.

## Quick start

Place these two files in the same directory:

```text
gaia_al_simulator/
├── download_gaia_spice.py
└── simulate_gaia_al.py
```

Install the required Python packages:

```bash
pip install numpy pandas spiceypy
```

Download the Gaia SPICE kernels from the official ESA SPICE Service:

```bash
python download_gaia_spice.py
```

On Windows, if `python` is not recognized, use:

```powershell
py download_gaia_spice.py
```

The downloader creates a local `Gaia/` directory next to `simulate_gaia_al.py`. It downloads the current Gaia operational meta-kernel `gaia_ops.tm` and the kernel files listed in that meta-kernel.

After the download, the directory should look like:

```text
gaia_al_simulator/
├── Gaia/
│   └── kernels/
├── download_gaia_spice.py
└── simulate_gaia_al.py
```

Then run the simulator:

```bash
python simulate_gaia_al.py
```

On Windows, if `python` is not recognized, use:

```powershell
py simulate_gaia_al.py
```

## Required inputs

### 1. Gaia SPICE kernels

The simulator requires Gaia spacecraft ephemeris information from the official ESA SPICE Service.

The Gaia SPICE kernels are downloaded automatically by `download_gaia_spice.py`; users do not need to download them manually.

Official ESA Gaia SPICE directory:

```text
https://spiftp.esac.esa.int/data/SPICE/GAIA/
```

The local copy of the operational meta-kernel is saved as:

```text
Gaia/kernels/mk/gaia_ops.tm
```

`download_gaia_spice.py` also updates the local `PATH_VALUES` entry in `gaia_ops.tm` so that it points to the downloaded `Gaia/kernels/` directory on your computer.

### 2. GOST scanning-geometry CSV

Set this path in `simulate_gaia_al.py`:

```python
GOST_CSV = r"/path/to/user_data/gost_TARGET.csv"
```

The file must contain the GOST columns used in `GOST_REQUIRED_COLUMNS`, especially:

```text
Target
ra[rad], dec[rad]
ObservationTimeAtBarycentre[BarycentricJulianDateInTCB]
scanAngle[rad]
parallaxFactorAlongScan
parallaxFactorAcrossScan
CcdRow[1-7]
Fov[FovP=preceding/FovF=following]
```

### 3. AL uncertainty curve, optional

If `USE_FIG_A1_AL_ERROR=True`, provide:

```python
GAIA_ALSIG_CSV = r"/path/to/user_data/DR3_ALsig.csv"
```

`DR3_ALsig.csv` gives a practical Gaia-like AL single-epoch uncertainty as a function of `G` magnitude, digitized from the blue AL field-angle residual scatter curve in Fig. A.1 of Lindegren et al. (2021).

If no magnitude-dependent error curve is used, set:

```python
USE_FIG_A1_AL_ERROR = False
SIGMA_AL_MAS = 0.20
```

## Reference point consistency

The AL values are expressed in a local tangent plane. Its reference point is defined in `simulate_gaia_al.py` by:

```python
STAR = dict(
    ra_ref_deg=...,
    dec_ref_deg=...,
    dalpha0_star_mas=0.0,
    ddelta0_mas=0.0,
    pmra_star_masyr=...,
    pmdec_masyr=...,
    parallax_mas=...,
)
```

With the usual setting

```python
dalpha0_star_mas = 0.0
ddelta0_mas = 0.0
```

`ra_ref_deg` and `dec_ref_deg` are the tangent-plane origin at `t_ref`.

For a self-consistent data set, these three coordinates must describe the same sky position:

1. the RA/Dec used to generate the GOST file;
2. `STAR["ra_ref_deg"]` and `STAR["dec_ref_deg"]` in `simulate_gaia_al.py`;
3. the reference point used later by the fitting code.

For Gaia DR3-style simulations, use the Gaia DR3 catalog position at J2016.0 for both the GOST query and the `STAR` block. Avoid relying on automatic SIMBAD name resolution when strict coordinate consistency is needed.

## Reference epoch

Default:

```python
USE_CUSTOM_T_REF_JD_TCB = False
REF_EPOCH_JYEAR_TCB = 2016.0
```

If a custom epoch is used, set:

```python
USE_CUSTOM_T_REF_JD_TCB = True
CUSTOM_T_REF_JD_TCB = ...
```

Then make sure the `STAR` parameters are defined at the same epoch.

## Key orbital conventions

- `pmra_star_masyr` is `mu_alpha*`; do not multiply by `cos(dec)` again.
- `omega_deg` is the companion-relative argument of periastron.
- The code uses a negative astrometric semimajor axis internally to generate the stellar reflex motion.
- Keep the same convention in the fitting code and in any comparison with literature values.

## Output files

Set these values in `simulate_gaia_al.py`:

```python
OUTPUT_DIR = r"/path/to/user_data/gaia_al_output_TARGET"
TARGET_NAME = "TARGET"
```

The script writes:

```text
*_gaia_al_raw_debug_j2016.csv      full diagnostic table
*_gaia_al_raw_matrix_j2016.txt     fitting matrix, whitespace-delimited
*_gaia_al_raw_matrix_j2016.dat     same matrix, .dat suffix
*_gaia_al_raw_matrix_j2016.csv     same matrix, comma-delimited
*_truth_j2016.json                 truth parameters and matrix metadata
```

The word `raw` is kept in the filenames for backward compatibility. The simulated quantity is still a local-plane AL observable, not a Gaia CCD crossing time.

## Compact matrix format

First row:

```text
[AL_start_row, N_al, floor(t_ref_jd_tcb), fractional_t_ref_jd_tcb, ...]
```

Each data row:

```text
obs_time_tcb
centroid_pos_al_mas
centroid_pos_error_al_mas
scanAngle_rad
sin_psi
cos_psi
parallaxFactorAlongScan
outlier_flag
synthetic_transit_id
ccd_row
fov_code
```

`synthetic_transit_id` is only a sequential label assigned by this script. `ccd_row` is the Gaia focal-plane row from GOST, not the AF strip number.

## Before running

Before running `simulate_gaia_al.py`, check:

1. the Gaia SPICE kernels have been downloaded under `Gaia/kernels/`;
2. GOST RA/Dec and `STAR` RA/Dec are the same reference point;
3. `STAR` parameters match `t_ref`;
4. the `omega_deg` convention matches the fitting code;
5. `INJECT_OUTLIERS=False` for clean likelihood validation.
